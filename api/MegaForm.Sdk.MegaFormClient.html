Class MegaFormClient

Namespace
MegaForm.Sdk

MegaForm.Sdk.dll

Default IMegaFormClient — a thin facade that maps SDK DTOs to/from MegaForm.Core repositories. It does NOT contain business logic; it reuses Core. Tenant/user context comes from the ambient MegaForm.Core.Interfaces.IPlatformContext (when the host provides one) and can be overridden per call via MegaFormScope.

public sealed class MegaFormClient : IMegaFormClient, IFormApi, ISubmissionApi, IDashboardApi, ISubmissionDashboardApi, IInboxApi, IFileApi, ISchemaApi

MegaFormClient(IFormRepository, ISubmissionRepository, IPlatformContext?, IFileRepository?, IStorageService?, SubmissionProcessor?)

Create a client. platform/files/storage are optional — when platform is null, every call must pass an explicit MegaFormScope; file APIs need files+storage. submissionProcessor is optional: when supplied (the host registered it in DI), SubmitAsync runs the FULL submission pipeline (validate + anti-spam + notify + workflow + index), identical to a public form submit; when null, it falls back to validate-then-insert.
