ARCHITECTURE OVERVIEW
============================================================
MegaForm.Core/          C# shared library (interfaces, models, services)
MegaForm.UI/            TypeScript project (ALL UI code — portable)
MegaForm.DNN/           DNN module (thin wrapper)
MegaForm.Oqtane/        Oqtane module (thin wrapper)
MegaForm.Umbraco/       Umbraco module (thin wrapper) [NEW]
MegaForm.Web/           Standalone ASP.NET Core
Assets/                 Shared static files (CSS, legacy JS, TS build output)

Goal: Each platform view = ~10 lines HTML (mount div + load JS bundle)
All UI logic lives in MegaForm.UI/ TypeScript project.

TECHNOLOGY STACK
============================================================
- TypeScript 5.4+ (strict mode)
- Vite 5 (build tool, HMR dev mode)
- Vanilla TS (no framework — zero runtime overhead)
- DOM helpers: custom h() function for element creation
- Output: IIFE bundles → Assets/js/bundles/
  - megaform-config.js  (Config Panel + Template Gallery)
  - megaform-builder.js (Form Designer SPA)
  - megaform-views.js   (List, Card, Detail, Continuous renderers)

DEV WORKFLOW
============================================================
- VS Code opens MegaForm.UI/ folder
- npm run dev → Vite watch → auto compile on save
- Output → Assets/js/bundles/ → platform picks up immediately
- Visual Studio opens .sln for C# — two IDEs in parallel
- npm run build → production build with minification + sourcemaps

PLATFORM-SPECIFIC NOTES
============================================================

DNN (Current — fully functional):
  - Auth: jQuery.ServicesFramework (TabId, ModuleId, Token)
  - API: /API/MegaForm/Controller/Action
  - Views: ASCX files (will become thin wrappers)
  - DB: Raw ADO.NET (FormRepository static methods)

Oqtane (Scaffolded — needs testing):
  - Auth: Blazor auth + Bearer token
  - API: /api/megaform/Controller/Action
  - Views: Razor components
  - DB: EF Core (EfRepositories.cs)

Umbraco 14+ (Scaffolded — needs testing):
  - Auth: X-UMB-XSRF-TOKEN cookie
  - API: /umbraco/api/megaform/Controller/Action
  - Views: cshtml partial views
  - DB: EF Core (shared DbContext pattern)
  - Integration: Composer for DI, PluginController for API
  - Backoffice: Consider Lit-based property editor for Bellissima

Standalone .NET (Scaffolded):
  - Auth: ASP.NET Core Identity / cookie
  - API: /api/megaform/Controller/Action
  - Views: Razor Pages
  - DB: EF Core


============================================================
API CONTRACT (shared across all platforms)
============================================================
All platforms expose identical endpoints, only prefix differs:

  GET  {base}/Form/Get?formId={id}
  GET  {base}/Form/List?moduleId={id}
  POST {base}/Form/Save
  POST {base}/Form/Delete?formId={id}
  GET  {base}/Form/Stats?formId={id}
  POST {base}/Form/Duplicate?formId={id}

  GET  {base}/Submit/Schema?formId={id}           (public)
  POST {base}/Submit/Post                          (public)

  GET  {base}/Submissions/List?formId={id}&...
  GET  {base}/Submissions/Get?submissionId={id}
  POST {base}/Submissions/UpdateStatus?...
  POST {base}/Submissions/Delete?submissionId={id}
  GET  {base}/Submissions/Export?formId={id}&format=json

  GET  {base}/ModuleConfig/Get?moduleId={id}
  POST {base}/ModuleConfig/Save
  GET  {base}/ModuleConfig/Fields?formId={id}


============================================================
BUILD SCRIPTS
============================================================

TS Build (MegaForm.UI/):
  npm run dev     → watch mode, outputs to Assets/js/bundles/
  npm run build   → production build with minification

DNN Package:
  BuildPackage-DNN.ps1 → copies Assets/ + builds DNN install zip

Oqtane Package:
  BuildPackage-Oqtane.ps1 → copies Assets/ + builds NuGet

Umbraco Package:
  BuildPackage-Umbraco.ps1 → copies Assets/ to App_Plugins/ + builds NuGet

Each build script should run "npm run build" in MegaForm.UI/ first.
